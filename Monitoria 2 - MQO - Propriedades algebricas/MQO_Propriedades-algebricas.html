1.2 PIPE (%>%).

O PIPE um novo operador adicionado pelo pacote magrittr, presente no tidyverse. Ele permite passar um objeto a uma funo de forma encadeada, o que torna o cdigo mais limpo. Ele faz isso pegando o objeto da esquerda e jogando no primeiro argumento da funo a direita. Vejamos um exemplo:

# Para escrever o PIPE, basta usar o atalho: ctrl + shift + m
10 %>% log()

10 %>% log(1,base = .)

2. Regresso simples - Estimando coeficientes

Supondo que queremos estudar a relao entre os salrios dos CEOs (salary) e o retorno mdio sobre o patrimnio lquido (roe). Para isso, podemos estimar a seguinte regresso simples:

\(salary = \beta_{0} + \beta_{1}roe + u\)

Vamos importar novamente os dados e selecionar as colunas que vamos utilizar:

dados <- read_csv("um_csv.csv") %>% 
  select(c("salary","roe"))

O primeiro passo em qualquer anlise economtrica a observao dos dados. Para isso, utilizaremos o pacote ggplot:

#help(ggplot)

Para plotar primeiro criamos um objeto ggplot, passando os dados e o aesthetics, que indica as variveis que queremos plotar. Nesse caso, queremos plotar um grfico de disperso com roe no eixo x e salary no y:

# passamos ento os dados para o primeiro argumento com o pipe, e o aesthetics dentro da funo aes(), indicando o eixo de cada varivel:
dados %>% 
  ggplot(aes(x = roe,y = salary))

O grfico saiu em branco pois ainda no informamos qual tipo de grfico ns queremos. Precisamos adicionar ao plot um geom, que ir indicar seu tipo. Como queremos um grfico de disperso, usaremos o geom_point():

dados %>% 
  ggplot(aes(x = roe,y = salary)) +
  geom_point()

Ao visualizar os dados, podemos observar que apenas trs observaes apresentam salrio maior que 5000. Para melhorar a regresso e facilitar a visualizao, iremos filtrar esses outliers da nossa amostra:

dados <- dados %>% 
  filter(salary < 5000)   # Filtrando apenas observaes onde salary < 5000

dados %>% 
  ggplot(aes(x = roe,y = salary)) +
  geom_point()

Satisfeitos com os dados, partimos para a estimao do nosso modelo. As frmulas a seguir nos do as estimativas dos parmetros:

\(\hat{\beta_{0}} = \bar{y} - \hat{\beta_{1}}\bar{x}\)

\(\hat{\beta_{1}} = \frac{\sum^n_{i=1} (x_{i} - \bar{x})(y_{i} - \bar{y})}{\sum^n_{i=1}(x_{i} - \bar{x})^2} = \frac{Cov(x,y)}{Var(x)}\)

As frmulas podem ser derivadas a partir do mtodo dos momentos ou do MQO, os dois mtodos chegam no mesmo resultado de formas diferentes. A derivao do mtodo dos momentos est na pgina 28 do Wooldridge.

Com isso, estimamos:

#Para facilitar, podemos nomear as variveis no nosso environment fazendo um subsetting:
salary <- dados[,"salary",drop = TRUE]
roe <- dados[,"roe",drop = TRUE]

#Estimando a inclinao:
cov_xy <- cov(roe,salary)
var_x <- var(roe)
hat_beta_1 <- cov_xy/var_x

#Intecepto:
hat_beta_0 <- mean(salary) - hat_beta_1*mean(roe)

as_vector(c("hat_beta0" = hat_beta_0,"hat_beta1" = hat_beta_1))
## hat_beta0 hat_beta1 
## 878.16147  15.30312

Portanto, a partir da amostra utilizada, a reta da regresso que relaciona os salrios e roe dada por:

\(\widehat{salary} = 878.16147 + 15.30312(roe)\)

Plotando a reta estimada:

plot1 <- dados %>% 
  ggplot(aes(y = salary,x = roe)) +
  geom_point() +
  geom_abline(intercept = hat_beta_0,slope = hat_beta_1,color = "red")

plot1

3. Interpretando os coeficientes.

Quando roe = 0, a estimativa de salrio igual ao intercepto, $878.16. Podemos calcular tambm o salrio mdio dado uma variao de roe:

\(\widehat{\Delta salary} = 15.30312(\Delta roe)\)

Ou seja, se roe varia em 1, o salrio mdio dos CEOs aumenta em $15.30.

# Escrevendo uma funo que calcule o salrio para qualquer valor de roe:
salary_ceo <- function(roe){
    hat_beta_0 + hat_beta_1*roe
}

roe_x <- c(1,2,3)

salary_ceo(roe_x)
## [1] 893.4646 908.7677 924.0708

6. Qualidade de ajuste.

A propriedades descritas nos permites separar o modelo MQO em duas partes:
1. Uma parte explicada, correspondente aos valores estimados, que depende de x
2. Uma parte no explicada, correspondente aos resduos, que independe de x.

  • Soma dos quadrados explicada:
    \(SQE = \sum^n_{i=1}(\hat{y_{i}} - \bar{y})^2\)
SQE <- sum((fitted - mean(salary))^2)
SQE
## [1] 3514457
  • Soma dos quadrados dos resduos:
    \(SQR = \sum^n_{i=1}\hat{u_{i}}^2\)
SQR <- sum(uhat^2)
SQR
## [1] 73064701
  • Soma dos quadrados total:
    \(SQT = \sum^n_{i=1}(y_{i} - \bar{y})^2\)
SQT <- sum((salary - mean(salary))^2)
SQT
## [1] 76579158

A variao total em \(y\) pode ser expressa como a soma da variao explicada e a variao no explicada:

\(SQT = SQE + SQR\)

as_vector(c("SQT" = SQT , "SQE + SQR" = SQE + SQR))
##       SQT SQE + SQR 
##  76579158  76579158

Essa decomposio nos permite calcular o quanto da variao total presente em \(y_{i}\) conseguimos explicar a partir da variao total em \(\hat{y_{i}}\):

\(R^2 \equiv SQE/SQT = 1 - SQR/SQT\)

Calculando:

rsquared <- 1 - SQR/SQT
rsquared
## [1] 0.04589313
